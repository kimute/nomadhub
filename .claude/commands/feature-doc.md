# 기능 문서화

프로젝트의 구조를 분석하고 기능 폴더 아래에 해당 기능을 정의하는 MD 파일을 자동으로 생성합니다.

## 작업 흐름

### 1. 프로젝트 구조 분석
- `src/` 디렉토리 구조 스캔
- 기능 관련 폴더 식별 (features, app 등)
- 각 기능별 파일 목록 수집

### 2. 기능 파일 분석
- 각 기능 폴더 내 TypeScript/JavaScript 파일 분석
- Export된 컴포넌트, 함수, 타입 추출
- Import 관계 파악

### 3. MD 파일 생성
각 기능 폴더에 `FEATURE.md` 파일을 생성하며 다음 내용 포함:

#### 구조
```markdown
# [기능명]

## 개요
- 기능 설명
- 목적 및 사용 사례

## 구조
- 폴더 구조
- 주요 파일 목록

## 컴포넌트/함수
- Export된 항목 목록
- 각 항목의 역할 설명

## 의존성
- 사용하는 외부 라이브러리
- 내부 모듈 의존성

## 사용 예시
- 코드 예제
- 통합 방법

## API (해당하는 경우)
- Endpoint 목록
- 요청/응답 형식

## 상태 관리 (해당하는 경우)
- 사용하는 상태
- 상태 흐름

## 개발 가이드
- 수정 시 주의사항
- 테스트 방법
```

## 실행 절차

1. **디렉토리 스캔**
```bash
find ./src -type d -name "features" -o -type d -name "app"
```

2. **각 기능 폴더 분석**
   - 폴더 내 파일 목록 확인
   - TypeScript 파일 읽기 및 AST 분석
   - Export, Import 문 추출

3. **문서 템플릿 작성**
   - 수집된 정보를 기반으로 MD 파일 생성
   - 각 기능 폴더에 `FEATURE.md` 저장

4. **검증**
   - 생성된 문서 확인
   - 누락된 정보 체크

## 작업 규칙

- **자동화**: 최대한 자동으로 정보 추출
- **일관성**: 모든 기능 문서가 동일한 형식 유지
- **명확성**: 기술적이지만 이해하기 쉬운 설명
- **완전성**: 모든 주요 정보 포함
- **유지보수성**: 업데이트하기 쉬운 구조

## 출력 위치

- 각 기능 폴더: `[기능폴더]/FEATURE.md`
- 예시:
  - `src/components/features/auth/FEATURE.md`
  - `src/components/features/dashboard/FEATURE.md`
  - `src/app/api/users/FEATURE.md`

## 병렬 처리

여러 기능 폴더가 있는 경우:
- 각 기능 폴더를 병렬로 분석
- 독립적인 서브에이전트 생성 (최대 5개)
- 동시 문서 생성으로 시간 절약

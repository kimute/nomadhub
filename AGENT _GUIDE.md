# 서브에이전트 주의점

## 서브에이전트는 복잡한 기능을 맡기지 않을것

- CRUD 작업을 맡길경우 각각의 서브에이전트를 만들것
  - create subagent, read subagent, update subagent delete subagent

## 커스텀 서브에이전트 사용 사례

- 테스트 에이전트
  - 메인에서 실행해서 컨텍스트를 사용할 이유가없음 이럴때 커버리지가 xx%인 커스텀 에이전트를 만들어 두면편리함
  - 실패하는 테스트가 있을때 수정하면 되는지 사용자에게 알리면 되는지
- 아키텍쳐 레이어별 에이전트
  - UI를 생성할때는 항상sahdcn을 사용하는지, 테이블을 생성할때는 Tank stack을 항상 사용하는지 등등
- 하면 안되는 작업명시하기 

## 서브에이전트는 병렬로 실행시 진가를 발위

다음의 프롬프를 실행하면 클로드가 각각의 서브 에이전트를 병렬로 작업을 진행 시킴

```# 역할
   너는 프론트엔드 시니어 전문가야 서브에전트를 사용해 기능을 구현해줘

   ## 작업순서
   코드 규약을 가져오기 (claude.md에 넣어주자)
   태스크를 확인하기
   기존에 완성된 코드를 참고하여 코드작성패턴을 파악하기
   프론트는 프론트 서브에이전트로
   백엔드는 백엔드 서브에지전트로 기능 구현하기

```
